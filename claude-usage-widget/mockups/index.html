Data & field map

Every visible value is sourced from one undocumented endpoint on the consumer claude.ai web app.

Endpoint

GET https://claude.ai/api/organizations/{orgId}/usage

Auth: sessionKey cookie. Org id from lastActiveOrg. Stored in EncryptedSharedPreferences on device.

Field map

Ring 1 — "5-hour" five_hour.utilization
Ring 1 reset time five_hour.resets_at
Ring 2 — "7-day Opus" seven_day_opus.utilization
Ring 2 reset time seven_day_opus.resets_at
Countdown value Computed from five_hour.resets_at − now
Countdown progress bar Computed from five_hour.utilization

Empty / null states

  • If seven_day_opus is null (free tier or no Opus quota), Ring 2 renders muted with an em-dash and label "no Opus on plan".
  • If the network call fails, last-known values render with a small ⚠ in the header and the countdown freezes.
  • If the cookie is invalid (HTTP 401/403), widget shows a "Tap to re-auth" CTA instead of the rings.

Refresh

  • WorkManager periodic job every 15 min (Android's minimum interval).
  • Tap-to-refresh on the widget body.
  • Forced refresh when phone screen turns on (ACTION_USER_PRESENT).

Out of scope (for now)

  • Active sessions / sessions awaiting input — not exposed on this endpoint, no documented public endpoint found.
  • Daily totals — API only exposes 5-hour rolling and 7-day windows.
  • Per-conversation drill-down.
